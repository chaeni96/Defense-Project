Introduction

Isle of Lore 2: UI Pack is a pack of user interface elements and icons for your video game project. They work great with my other Isle of Lore 2 packs, including Isle of Lore 2: Hex Tiles!

The images come as png files for direct use, as well as Krita (.kra) and Photoshop (.psd) source files for your personal edits. The pack is compatible with my batch image editing tool Color Mapper, which means you can easily change the color palette of the images to your own liking... the required Color Mapper Files are included!

This documentation will go into more detail on how to find the files you want and how to change and re-export them to fit your needs.

Please keep the license in mind, which is available as a License.txt-file in the folder you downloaded. If you have questions or feedback, you can reach me through the following channels. Content Structure

The asset pack contains a folder called Sources and in it, you'll find the three folders input, output and misc. While misc contains lists of names of various keyboard keys and gamepad buttons, it also includes a dummy character portrait which you can use as a placeholder. The input and output folders are the ones relevant for a Color Mapper setup. If you don't use Color Mapper, you can head straight to the output-folder and use the files there, but if you do, please refer to the section Re-exporting the Images with Color Mapper further down.

Let's check the output-folder. All icons and control icons (like gamepad buttons) are available in the ui_pack_controls-folder within there, with gamepad-related icons sorted into ui_pack_gamepad_xxx-folders, keyboard-related icons into ui_pack_keyboard_xxx-folders and mouse-related icons into ui_pack_mouse_xxx-folders. There are also symbol icons with and without a dark outline. Both variants have their respective folders ui_pack_icons.standard and ui_pack_icons.outline.
The interface elements on the other hand are available in the ui_pack_elements-folder within the output-folder. The section Interface Elements will go into more detail on how these files are used.

After we had a look at the output-folder, let's check the content structure of the input\ui_pack_controls and input\ui_pack_elements-folder, where you'll find all the Krita (.kra) and Photoshop (.psd) files as well as image file exports (.png) of all the single layers of every icon. These files are required if you want to use Color Mapper. The Color Mapper Files *.colormapper.txt tell Color Mapper how to assemble the icons.

Interface Elements

The interface elements are available in Sources\output\ui_pack_elements. Some interface elements are tileable, which means they come in a small size with their bounds fitting a 3x3 grid. If so, the documentation gives a declaration like 3x3 16x16px, which means the texture is tileable by 3x3 with every tile being 16x16px in size. This is also called 9-slicing and some game engines have the capability out-of-the-box. If not, you have to render the 4 corners separately and repeat the borders and the inner tile accordingly.

Shadows

Some elements have a drop shadow around them, which is built in. You can easily remove that shadow with the help of Color Mapper. Just open the Color Mapper files of the elements whose shadow you want to have removed, for example

.\input\ui_pack_elements\heart_piece\standard.colormapper.txt

, and remove the line *shadow -> iol2_dark from the file, then re-export the icons with Color Mapper again and the shadow should be gone.

An exception are box elements like the dialog box or the tooltip, which have a separated shadow, as you have to place tabs or arrows with their own shadows in front of the box' shadow, but behind that box. If you want to use the dialog box without tabs though, there is a version with baked-in shadow for convenience nonetheless.

Overall, separating overlaying interface elements with some sort of shadow from the game world is strongly recommended, as otherwise, the elements may merge with the game world too much, leading to more effort for the player to identify interface elements on the screen properly.

Dialog Box

The dialog box and its parts are available in folders starting with dialog. It comes in two variants: with a shadow already baked in (dialog_box_with_shadow) or the panel without a shadow (dialog_box.standard) and a separate shadow (dialog_box_shadow.standard) to put behind the dialog box. If overlaying the dialog box over the game world, I would strongly recommend to use a shadow, either the one baked in, by placing the separated shadow behind it or by using your game engine's drop shadow capabilities. The shadow's top-left corner can be displaced by (-40, -40) pixels or less from the dialog box' top-left corner. The following image hopefully clarifies how to use the dialog box.

The dialog box without a shadow is tileable by 3x3 16x16px, while the dialog box shadow and the dialog box with shadow are tileable by 3x3 96x96px.

Assembling the dialog and its shadow from scratch vs using the dialog box with its shadow baked in.

A gray square can be found in dialog_box_top_right_bg.standard and it is meant to be placed in the top-right corner of the dialog box in case you want to place a button onto it, like the following image shows. It is tileable by 3x3 16x16px and has to be extended depending on how thick you want the gray area to be around the button.

Placing a gray box and then a button in the top-right corner of the dialog box.

the gray box at dialog_box_top_right_bg.standard can also be used as an in-dialog inventory slot, like shown below.

Placing a gray box as an inventory slot.

Dialog boxes can be extended by tabs, which are available in dialog_box_front_tab.standard and dialog_box_back_tab.standard respectively. Make sure to place them in a way so they connect to the black border line of the dialog box. The front tab has a white area that overlaps the gray decorative border of the dialog box. Use the separate dialog box shadow dialog_box_shadow.standard to generate an additional shadow behind the tabs.

Front and back tabs placed next to the dialog box, with an additional shadow behind them.

Conversation Box

The conversation box has to be assembled from multiple parts to achieve the effect shown in the image below. The box.standard-folder contains both a white box tileable by 3x3 10x10px and a box outline with transparent content tileable the same way. The box outline can be placed on top of character portraits. The folders box_arrow.standard and box_arrow_shadow.standard contains the big arrow and its shadow, with the latter being placed below both the arrow and the box it is connecting to. In name_plate.standard you'll find a nameplate tileable by 3x3 18x18px. It is best placed behind the box and moved down a bit with the bottom round corners not visible anymore. The shadow in box_shadow.standard can be used as both a generic shadow for rectangular elements as well as for the portrait, the nameplate and the box itself. The usual shadow displacement is (-40, -40) pixels.

A conversation box assembled from different parts. The portrait is of exemplary nature and available in the folder Sources\misc\example_portrait.

Speech Bubble

Speech bubbles are little text or icon windows placed in the game world, for example above the head of a non-player character. They are available in two variants (with an outline and without) in speech_bubble.standard, with the speech bubble arrows and their shadows in speech_bubble_arrow.standard and speech_bubble_arrow_shadow.standard. When using the speech bubble with a border, make sure that the arrow's white area connects with the white area of the speech bubble, like shown in the image below. Also note that the arrow's shadow has to be placed behind the speech bubble. The shadow displacement is (-8, -8) pixels.

A speech bubble with a dark outline (left) and one without (right).

Tooltip

The tooltip boxes shown in the image below can be used as little hints when placing the mouse on a button and waiting for a second. The tooltip box itself can be found in the folder tooltip.standard and is tileable by 3x3 24x24px, with its shadow from the same folder tileable in the same manner. The shadows are displaced by (-8, -8) pixels relative to the box. There are two types of arrows: an orthogonal version in the folders tooltip_arrow.standard (with its shadow in tooltip_arrow_shadow.standard) as well as a diagonal one in the folder tooltip_arrow_diagonal.standard (with its shadow in tooltip_arrow_diagonal_shadow.standard).

A tooltip with an arrow at the top and a tooltip with a diagonal arrow at the top-left. The shadows of both the box and the arrows have to be placed both behind the box and the arrow.

Scrollbar & Slider

Scrollbars and sliders can be found in the folders whose names start with scrollbar. There is quite some variation available in the bars and buttons. For the bar, use either a bar (scrollbar_horizontal_bar.standard) or a line (scrollbar_horizontal_line.standard), both of them tileable by 3x1 12x12px respectively 3x1 4x4px. Then place a diamond (scrollbar_diamond.standard), a stick (scrollbar_horizontal_stick.standard) or a button (scrollbar_horizontal_button.standard) on top. All these button types have a pressed variant in the corresponding folder containing the keyword pressed. The bars come in a blue and a white variant, and both filled and unfilled, which can be put together to create a slider. Check the following image for a visualization.

Re-exporting the Images with Color Mapper

Please make sure to use the most up-to-date version of Color Mapper, as this asset pack requires new colors added to the config.

If you want to use the images the way they are, refer to the Sources\output-folders. But if you want to change the images, for example replace some or all of the colors, you can use my batch image editing tool Color Mapper. It can also re-size, re-scale and re-position icons, all in one go.

Color Mapper assembling a button from its single parts.

Color Mapper assembling a button, but also switching out some of the colors.

To make Color Mapper work, the Isle of Lore 2: UI Pack download includes the folders Sources\input\ui_pack_controls and Sources\input\ui_pack_elements, containing the Krita (.kra) and Photoshop (.psd) source files as well as png exports of the single layers and Color Mapper Files (.colormapper.txt) containing information on how Color Mapper is going to assemble the files.

Color Mapper isn't part of this asset pack and instead has to be downloaded here, which allows me to release updates for Color Mapper independently of the projects it is used in. The download includes a Documentation-folder with exact instructions on how to use Color Mapper, but the workflow in general is at follows:

  1. Setup Color Mapper's configuration file so it knows where to find the Sources\input and Sources\output folders.
  2. Running Color Mapper, which will gather all the *.colormapper.txt-files in the input folder and assemble the png files from the single layer files with colors referenced in the *.colormapper.txt-file and defined in Color Mapper's configuration file. The assembled png files will be put in the output-folder.
  3. After checking the results, the files from the output-folder are ready to use.

Design Notes

  • The icons were drawn in Krita with the Ink-2 Fineliner.
  • The border size for big interface elements is 12px, for smaller elements 6px.
  • The size of text written on the control images like keyboard keys is 21px.
  • Symbol icons were drawn with a 10px thick line.
  • Lines for symbols on keyboard controls are either 17px or 13px thick.
  • Lines for symbols on gamepad controls are either 16px or 10px thick.
  • A lot of elements have rounded borders. The roundness is usually 12px, 24px or 64px. Special cases are the creation of the mouse image, which has a roundness of 50px and keyboard keys, which have a roundness of 32px. The dialog box uses a radius of 12px for the outer line and 9px for the decorative line inside.

Shadows were created by selecting the opaque of the whole image and filling it with the dark color #161616 in a new layer (basically creating a silhouette). Then, the "Blur" filter (horizontal and vertical radius see below, strength 0) was applied. In some cases, the shadow was duplicated and merged to create a stronger shadow, and sometimes the layer itself got a reduced opacity for making it less pronounced overall.
